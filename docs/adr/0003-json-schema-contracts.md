# ADR 0003: JSON Schema Contracts

Status: accepted

## Context

Rewrit needs portable contract files that can be written by humans, generated by
tools, reviewed in pull requests, and validated in CI.

Contracts must describe behavior without depending on one framework's test API.

## Decision

Contracts are JSON-compatible documents with versioned schemas generated from
the Rust model types.

The current contract schema version is `rewrit.contract.v1`. Contract files may
be JSON and can be represented in YAML/TOML-derived tooling as long as the final
shape matches the schema.

The CLI exports schemas with:

```bash
cargo run -p rewrit-cli -- schema export --kind all --out-dir dist/schemas
```

## Consequences

Positive:

- Contracts are language-neutral.
- Editors and CI can validate contract shape.
- Schema changes are reviewable.
- Agent and dashboard tooling can rely on a stable machine format.

Tradeoffs:

- Contract expressiveness is constrained by the public schema.
- Advanced framework-specific behavior belongs in adapters, not in the contract
  language.
- Schema changes require versioning discipline.

## Follow-Up Rule

Public protocol, report, or contract shape changes should update generated
schemas and, when architectural intent changes, add or update an ADR.
